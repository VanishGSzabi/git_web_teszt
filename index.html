Rövid bemutató

Mi az a git?

A git az egy ingyenes, nyílt forráskódú verziókezelő szoftver, amely egy fájlállomány összes változását nyomon követi. A mentett verziókat vissza lehet hívni, és onnan folytatni a munkát. Amíg a git csak helyi verziókezelési feladatokat lát el, hatékonyságát tovább lehet növelni távoli elérésű tárolókkal mint a Github. A szoftvert egyéni felhasználóktól multimilliós cégek is használják, egyszerű beadandóktól akár egy egész operációs rendszer fejlesztéséig használják.

Mire jó a git?

A git használásával projektek tárolása és változásainak követése végtelenül hatékonyabb lesz. A tárolt verziókat (ha nem lettek törölve) bármikor elérhetjük, így a projekt menetét követhetjük, visszaállíthatjuk.

Git telepítése, beállítása és projekt kezdése

Letöltés

A git elérhető Windows-ra, macOS-re illetve több Linux disztribúciókra (legtöbb asztali disztroban elöre telepítve.)

Beállítás

A git parancsként működik a Windows cmd-ben (parancssorban). Ellenőrizhetjük a szoftver telepített verzióját a git --version paranccsal, illetve a git --help paranccsal egy grafikus súgó nyitható meg. A hatékonyság kedvéért állítsuk egy felhasználónevet és elérhetőséget a programba:

  • Felasználónév beállítása: git config --global user.name "nev"
  • Elérhetőség (email cím) beállítása: git config --global user.email "email@cim"

Projekt kezdése

Készítsünk egy mappát a projektnek és nyissunk ott meg egy parancssort: shift+jobb click -> Parancssor megnyitása itt. Powershell ablak esetén írjuk be, hogy cmd, és onnan folytassuk a munkát.

A git init paranccsal készítsük fel az állományt verziókezelésre.

Jelzés, tárolás, visszavonás

Fájlok jelzése

A git add paranccsal jelölni tudjuk, hogy mely mappákat, fájlokat szeretnénk verziókezelés alá rendelni. Érdemes az összes fájlt jelezni az állományunkban a git add *.* paranccsal, de egyenként is tudunk jelezni fájlokat, például:

  • csak egy adott fájlt: git add index.html - csak az index.html fájl lesz jelezve.
  • az összes megadott kiterjesztésű fájl: git add *.png - minden png fájl jelezve lesz.
  • az összes fájl amelyben a ? előtti vagy utáni karakterek következnek: git add ?_template.* - az összes _template-vel végződő fájl jelezve lesz.
  • egy mappa és azon belüli összes fájl jelzése: git add asset\bootstrap - az asset\bootstrap egész tartalma jelezve lesz.

Verzió tárolása

Ha végeztünk a módosításokkal, az állományt a git commit -m "megjegyzes" paranccsal tudjuk verzióként tárolni. Az összes tárolt verzió megjelenítéséhez a git log parancsot tudjuk használni.

Megnézhetjük, hogy a jelzett állományból mi került módosításra a git status paranccsal.
 Piros színnel jelennek meg az elemek nevei, ha tárolva vagy jelezve sem voltak, vagy csak módósítva lettek.
Zöld színnel jelennek meg az elemek nevei, ha jelezve lettek, de nincsenek tárolva.

Jelzés visszavonása

A git reset paranccsal vissza tudjuk vonni a megadott állományok jelzését, így nem lesznek tárolva.

  • csak egy adott fájlt: git reset index.html
  • az összes megadott kiterjesztésű fájl: git reset *.png
  • az összes fájl amelyben a ? előtti vagy utáni karakterek következnek: git reset ?_template.*
  • egy mappa és azon belüli összes fájl jelzésének visszavonása: git reset asset\bootstrap

Különbségek megtekintése

Különbségek állományban

Megtekinthetőek a változtatások a verziótárolt állomány és a jelenlegi között a git diff paranccsal.

  • Ha egy állomány változásait akarjuk megtekinteni: git diff index.html
  • Ha az összes módosítást akarjuk megtekinteni git diff HEAD

Tárolt verziók közötti különbség

A git show paranccsal meg lehet tekinteni egy commit és az azelőtti commit közötti különbséget.

  • Ha a legfrisebb commit és az azelőtti között: git show HEAD
  • Egy kiválasztott commit és az azelőtti között: git show commit_azonositoszama

Grafikus felület

A parancssoros munka helyett elérhető egy GUI amiben kicsit szofisztikáltabban lehet megtekinteni az teljes állományunk összes verzióját. Ehez a gitk parancs kell.
